## 1. 4개 질문과 답

### Q1 — 누구에게 가장 먼저 보여주고 "내 액이네" 느끼게 할까?
**답**: 신임 팀장 · "팀원이 날 무시하는 거 같아" 다루기

→ MANIFESTO 제1축(인지왜곡 = 과잉 일반화) 직격. 타겟이 **팀장 후보가 아니라 현직 신임 팀장**이고, 트리거는 **팀원과의 관계에서 오는 자기 회의**임을 확정.

### Q2 — 그 사람이 지금 그 생각이 들 때 실제 행동은?
**답**: 혼자 술 한잔 · 동기랑 카톡으로 퓸이

→ **현재 행동은 재구조화 없는 분출**. 우리 wedge는 술/카톡과 경쟁하면서 *한 발 더* — 분출을 받아주되 자기 인지로 끌고 가는 것. 단, CBT 원칙(되묻기) 때문에 "AI가 답을 주는" 방식은 못 씀.

### Q3 — 30초 안에 "오, 술/카톡보다 낫네" 만드는 화면은?
**답**: 카테고리 + **그 카테고리만의 첫 질문(가이드 질문)**

→ **v2 plan을 한 단계 진화시키는 가장 큰 발견**. dropdown만으로는 부족하다. 카테고리 선택 시 입력란 placeholder/첫 질문이 **그 카테고리에 특화된 한 줄**로 바뀌어야 사용자가 "이건 리더용이네" 즉시 인식.

### Q4 (Premise Challenge) — 두 번째 방문 이유는?
**답**: 익명 리더 커뮤니티 — "보니 나만 그런 게 아니었네"

→ 솔직한 인정: 이 HW 범위에서는 retention 답을 완성할 수 없다. **이번 feature는 첫 방문 임팩트에 집중**, 그 다음 자연스러운 진화는 MANIFESTO 제2축(익명 커뮤니티).

## 2. Biggest Lesson

> **dropdown은 라벨이고, placeholder는 정체성이다.**
>
> v2 plan은 카테고리 select만으로 정체성 간극을 메우려 했지만, 사용자가 첫 입력란에 손을 대는 그 순간이 "이건 리더용이네"가 결정되는 진짜 순간이다. 정체성은 metadata가 아니라 **사용자가 마주치는 첫 카피**에서 만들어진다.

부수 lesson:
- 우리 진짜 경쟁자는 다른 앱이 아니라 **술과 카톡이다** (Status quo).
- 인지왜곡 라벨 자동 표시 같은 "AI가 진단하는" 옵션은 매력적이지만 CBT 안전장치(진단 금지)와 정면 충돌 — 의식적으로 보류해야 함.

## 4. 선택한 액션: A1

### 정의 (v2 plan 대비 진화)

| 단계 | v2 plan | 이번 office-hours 후 |
|---|---|---|
| UI | 카테고리 select | 카테고리 select **+ 카테고리별 입력 placeholder/가이드 한 줄** |
| 데이터 | `category` 필드 | `category` 필드 (동일) |
| 프롬프트 | user 블록 prefix 1~2문장 | user 블록 prefix 1~2문장 (동일) |

### 카테고리별 placeholder 초안 (5개)

| 카테고리 | 첫 입력란 placeholder |
|---|---|
| 성과 압박 | "오늘 어떤 숫자/평가가 가장 무겁게 느껴졌나요?" |
| 팀원 관리 | "오늘 팀원 누구의 어떤 행동이 신경 쓰였나요?" |
| 평가·고과 | "구체적으로 어떤 평가 상황이 떠오르나요?" |
| 상사·보고 | "보고에서 가장 떨렸던 한 순간을 적어보세요" |
| 팀 내 갈등 | "누구와의 어떤 장면이 머리에서 안 떠나나요?" |
| 기타 | "지금 머릿속에 가장 먼저 떠오르는 한 줄을 적어보세요" |

### 왜 이 액션인가

1. **정체성 간극을 가장 직접 해소**: dropdown은 흘려보지만 placeholder는 반드시 본다.
2. **CBT 원칙 그대로 보존**: AI가 진단하지 않고 *질문* 형태로 사용자 자기 입력을 유도 — 되묻기 원칙과 정확히 부합.
3. **구현 비용 거의 0**: 정적 매핑 1개 객체 + placeholder 바인딩 한 줄. payload·프롬프트와 자연스럽게 연결.
4. **Status quo와 경쟁 가능**: 술·카톡이 못 주는 "구체적 자기 질문" 제공.
